zcp.json Specification

Configuration reference for the Zamp Control Plane deployment manifest

Download JSON Schema Add "$schema" to your zcp.json for editor autocomplete

Quick Start

{
  "name": "myapp",
  "services": [
    { "id": "maindb", "type": "postgres" },
    { "id": "cache",  "type": "redis" },
    {
      "id": "api",
      "type": "web",
      "runtime": "python",
      "basePath": "backend",
      "start": "gunicorn app:app --bind 0.0.0.0:5001 --workers 2",
      "port": 5001,
      "env": [
        { "name": "DATABASE_URL", "fromService": { "id": "maindb", "value": "connectionString" } },
        { "name": "REDIS_URL",    "fromService": { "id": "cache",  "value": "connectionString" } }
      ]
    },
    {
      "id": "kratos",
      "type": "container",
      "image": "oryd/kratos:v1.3.1",
      "command": "serve --dev --config /etc/config/kratos/kratos.yml",
      "port": 4433,
      "configFiles": { "/etc/config/kratos/kratos.yml": "kratos/kratos.yml" },
      "env": [
        { "name": "DSN", "fromService": { "id": "maindb", "value": "connectionString" } }
      ]
    }
  ]
}

Top-Level Fields

Field Type Required Description
name string required Application name. Combined with org slug for the Modal app name.
services Service[] required Array of infrastructure and compute service definitions.

Service Types

All resources are entries in the services array, discriminated by the type field. Infra services (postgres, redis) are provisioned first. Source-based compute (web, worker) deploys to Modal. Pre-built images (container) and Redis deploy to Fly.io. All compute services can reference infra outputs via fromService.

postgres

Infrastructure

Provisions a serverless Postgres database. Outputs are available to compute services via fromService.

Field Type Required Description
id string required Unique identifier used in fromService references.
type "postgres" required Must be "postgres".
schema string optional SQL to run after provisioning (e.g. CREATE TABLE statements).

Output Fields

Available via fromService.value in compute service env vars.

Value Description
connectionString Full Postgres connection URI (postgresql://...)
project_id Project ID
database_name Database name

Example

{ "id": "maindb", "type": "postgres" }

redis

Infrastructure

Provisions a persistent Redis instance on Fly.io with TLS and password auth.

Field Type Required Description
id string required Unique identifier used in fromService references.
type "redis" required Must be "redis".

Output Fields

Value Description
connectionString Full Redis URI (rediss://:<password>@<host>:<port>)
host Redis endpoint hostname
port Port number (as string)
authToken Redis password

Example

{ "id": "cache", "type": "redis" }

web

Compute

A compute service with a public HTTP endpoint, deployed to Modal via @modal.web_server.

Field Type Required Default Description
id string required — Unique identifier. Also used as the Modal function name.
type "web" optional "web" Service type. Defaults to "web" if omitted.
runtime string optional auto-detected python · nodejs · nextjs · docker · go
basePath string optional service id Directory containing service source, relative to zcp.json.
start string required — Shell command to start the service.
port integer optional 8000 Port the service listens on.
scaling Scaling optional {"min":1,"max":3} Autoscaling bounds. See Scaling.
env EnvVar[] optional [] Environment variables. See Environment Variables.

Example

{
  "id": "api",
  "type": "web",
  "runtime": "python",
  "basePath": "backend",
  "start": "gunicorn app:app --bind 0.0.0.0:5001 --workers 2",
  "port": 5001,
  "scaling": { "min": 1, "max": 3 },
  "env": [
    { "name": "DATABASE_URL", "fromService": { "id": "maindb", "value": "connectionString" } }
  ]
}

worker

Compute

A background process with no HTTP endpoint. Runs as a blocking subprocess via @app.function on Modal.

Field Type Required Default Description
id string required — Unique identifier.
type "worker" required — Must be "worker".
runtime string optional auto-detected Same runtimes as web.
basePath string optional service id Directory containing service source.
start string required — Shell command to start the worker.
scaling Scaling optional {"min":1,"max":3} Autoscaling bounds.
env EnvVar[] optional [] Environment variables.

Example

{
  "id": "temporal-worker",
  "type": "worker",
  "runtime": "python",
  "basePath": ".",
  "start": "python -m apps.workflows.worker",
  "scaling": { "min": 1, "max": 1 }
}

container

Compute — Fly.io

Deploy a pre-built Docker image from any registry (Docker Hub, GHCR, etc.) to Fly.io. No source code or Dockerfile needed.

Field Type Required Default Description
id string required — Unique identifier.
type "container" required — Must be "container".
image string required — Registry image reference. E.g. oryd/kratos:v1.3.1.
command string optional image default Override Docker CMD (appended to ENTRYPOINT). E.g. serve --config /etc/config/app.yml.
port integer optional — If set, expose as a public HTTPS endpoint on *.fly.dev. If omitted, run as a background worker.
configFiles object optional {} Map of {container_path: local_path}. Files are injected into the machine at boot.
scaling Scaling optional {"min":1,"max":3} Autoscaling bounds.
env EnvVar[] optional [] Environment variables. Supports fromService references.

Example — Ory Kratos (identity server)

{
  "id": "kratos",
  "type": "container",
  "image": "oryd/kratos:v1.3.1",
  "command": "serve --dev --config /etc/config/kratos/kratos.yml",
  "port": 4433,
  "configFiles": {
    "/etc/config/kratos/kratos.yml": "kratos/kratos.yml"
  },
  "env": [
    { "name": "DSN", "fromService": { "id": "maindb", "value": "connectionString" } }
  ]
}

Scaling

Field Type Default Description
min integer 1 Minimum containers. Set to 0 for scale-to-zero.
max integer 3 Maximum containers.

Environment Variables

Each entry in the env array is either a static value or a cross-service reference.

Static Value

{ "name": "FLASK_ENV", "value": "production" }

Service Reference

{ "name": "DATABASE_URL", "fromService": { "id": "maindb", "value": "connectionString" } }

fromService Reference

Field Type Description
id string The id of the infra service to reference.
value string The output field to read. See the output tables for each infra service type.

Runtime Auto-Detection

When runtime is omitted, ZCP detects it from the service's basePath directory:

File Found Detected Runtime
Dockerfile docker
package.json with next dependency nextjs
package.json without next nodejs
requirements.txt or pyproject.toml python
go.mod go

Next.js Two-Pass Deploy

When a manifest contains both a non-Next.js web service and a nextjs service, ZCP uses a two-pass deploy strategy:

  1. Pass 1: Deploy non-Next.js web services only. Capture the API URL.
  2. Pass 2: Deploy all services with NEXT_PUBLIC_API_URL injected into the Next.js image build.

This ensures NEXT_PUBLIC_* env vars are available at build time (baked into the static bundle by Next.js).
