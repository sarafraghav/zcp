The zcp.json manifest

Everything your app needs lives in one file. Databases, caches, and compute services — each is an entry in the services array.

zcp.json 
{
  "name": "myapp",
  "services": [
    { "id": "maindb", "type": "postgres" },
    { "id": "cache",  "type": "redis" },
    {
      "id": "api",
      "type": "web",
      "runtime": "python",
      "start": "gunicorn app:app --bind 0.0.0.0:5001",
      "port": 5001,
      "env": [
        { "name": "DATABASE_URL", "fromService": { "id": "maindb", "value": "connectionString" } },
        { "name": "REDIS_URL",    "fromService": { "id": "cache",  "value": "connectionString" } }
      ]
    }
  ]
}
postgres infra

Serverless Postgres database.

redis infra

In-memory cache.

web compute

Public HTTP endpoint.

worker compute

Background process, no HTTP.

container compute

Pre-built Docker image.

Key concept: fromService

Compute services reference infra outputs using fromService. For example, { "fromService": { "id": "maindb", "value": "connectionString" } } injects the Postgres connection URI as an env var at deploy time.
